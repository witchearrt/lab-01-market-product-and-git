# My role and skills

## My role

**Role:** Database Engineer

**Why I chose this role:**

1. I enjoy working with data structures and optimizing how information is stored and retrieved efficiently.
2. The role combines problem-solving with practical impact—database performance directly affects user experience.
3. I have foundational programming skills (Python, Java, C) that provide a solid base for database development and optimization.

## Skills I already have

1. **Git** — intermediate: comfortable with version control, branching, merging, and collaborative workflows.
2. **Python** — intermediate: can write scripts for data processing, automation, and basic database interactions (e.g., using SQLAlchemy, psycopg2).
3. **Java** — intermediate: understand object-oriented design, can work with JDBC and enterprise applications.
4. **C** — beginner: understand memory management and low-level concepts useful for database internals.
5. **SQL** — beginner: basic understanding of queries, joins, and table design from coursework and personal projects.

## Skills I clearly lack

1. **Advanced SQL & Query Optimization** — critical for writing efficient queries and tuning database performance. Essential for any Database Engineer role.
2. **Database Internals** — understanding storage engines, indexing strategies, and transaction management. Important for designing and maintaining robust database systems.
3. **NoSQL Databases** (MongoDB, Cassandra, Redis) — increasingly important for handling diverse data models and scaling requirements.
4. **Cloud Database Services** (AWS RDS, Aurora, Google Cloud SQL) — most companies now use cloud infrastructure; knowledge of managed database services is essential.
5. **Monitoring & Performance Tuning** (Prometheus, Grafana, slow query logs) — necessary for identifying bottlenecks and ensuring database reliability in production.

**Most challenging to acquire:** Database Internals—this requires deep theoretical knowledge and hands-on experience with database source code or building simplified database systems.

## My skills and market demands

Looking at the job postings, I notice that SQL and Python appear consistently—these are clearly foundational skills for database-related roles. I already have Python and Git, which is a good starting point, but my SQL skills need significant improvement to meet market expectations.

The demand for PostgreSQL specifically is notable; most postings mention it directly rather than just "SQL." This tells me I should focus on a specific database system rather than staying generic. Cloud experience also appears frequently, either directly (AWS, GCP) or indirectly through Docker and Kubernetes.

I'm concerned about my lack of practical experience with database administration tasks like backup/recovery, replication, and monitoring. These seem essential for real-world roles but aren't covered in my current coursework.

The presence of NoSQL databases in several postings suggests I shouldn't focus exclusively on relational databases. Redis appears often, often for caching, which seems like a practical skill to acquire.

Overall, I feel my programming foundation is solid, but I need to build database-specific expertise to be competitive.

## Skills that I can improve

**Primary focus: Advanced SQL and PostgreSQL**

- **Why:** SQL appears in nearly every posting, and PostgreSQL is the most frequently mentioned database. Strong SQL skills are foundational for any database role.
- **Resources:** PostgreSQL official documentation, "SQL Performance Explained" by Markus Winand, hands-on practice with LeetCode SQL problems or HackerRank.

**Secondary focus: Docker and basic cloud deployment**

- **Why:** Containerization appears in multiple postings, and cloud platforms are increasingly standard. Docker is a practical entry point.
- **Resources:** Docker official tutorials, deploying a PostgreSQL instance on AWS RDS free tier, following YouTube tutorials on containerizing database applications.

**Timeline:** One semester is realistic for achieving intermediate proficiency in SQL/PostgreSQL and basic competency with Docker and cloud database services.
